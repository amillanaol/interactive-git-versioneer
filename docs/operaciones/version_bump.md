# Version Bump

Incremento de número de versión en SemVer.

## Qué es Bump

**Bump** = incrementar la versión. El término proviene de "to bump up" (subir/levantar).

## Tipos de Bump

| Tipo | Cambio | Ejemplo |
| :--- | :--- | :--- |
| **patch** | Bug fixes | 1.2.3 → 1.2.4 |
| **minor** | Nuevas features | 1.2.3 → 1.3.0 |
| **major** | Breaking changes | 1.2.3 → 2.0.0 |

## Ejemplos de Bump

```
patch:  v1.0.0 → v1.0.1  (corregir bug)
minor:  v1.0.0 → v1.1.0  (nueva funcionalidad)
major:  v1.0.0 → v2.0.0  (cambio incompatible)
```

## Cuándo Hacer Bump

| Situación | Tipo |
| :--- | :--- |
| Bug fix menor | patch |
| Nueva funcionalidad compatibles | minor |
| API incompatible | major |
